┌────────────────────────────────────────────────────────────────┐
│ 🚀 WHICH LAUNCHER TO USE?                                      │
└────────────────────────────────────────────────────────────────┘

  FIRST TIME / HAVING PROBLEMS?
  → launch_gui_debug.bat
    Shows all errors, keeps window open

  DAILY USE (after confirming it works):
  → launch_gui.vbs
    Silent, no console window

  NEED TO SEE STATUS?
  → launch_gui.bat
    Shows brief status messages

┌────────────────────────────────────────────────────────────────┐
│ ⚙️ CONFIGURATION FILES                                         │
└────────────────────────────────────────────────────────────────┘

  env_config.json
  └─ Python environment settings
     • Which env to use (conda, venv, system)
     • Environment name (e.g., "aisa_env")
     • Activation command
     • Console visibility

  scrcpy_config.json (auto-created)
  └─ Scrcpy application settings
     • Scrcpy directory path
     • Default device IP/port
     • ADB path

┌────────────────────────────────────────────────────────────────┐
│ 🔧 TROUBLESHOOTING STEPS                                       │
└────────────────────────────────────────────────────────────────┘

  GUI won't start?
  1. Run: launch_gui_debug.bat
  2. Look for [ERROR] messages
  3. Follow the suggestions shown

  Need deep diagnostics?
  1. Run: test_setup.bat
  2. Check which test fails
  3. Fix the failed component

  Common fixes:
  • conda install tk          (if tkinter missing)
  • Edit env_config.json      (if wrong environment)
  • Check scrcpy path         (in GUI Settings tab)
